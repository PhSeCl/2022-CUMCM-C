# 问题一：分类变量关联分析设计

## 目标与范围

为问题一的前半部分建立探索性分析：考察 `纹饰`、`类型`、`颜色` 与 `表面风化` 四个分类变量的两两关联，为后续选择决策树或其他分类模型提供依据。

本阶段只描述样本中的关联结构，不训练预测模型，不选择决策树阈值，也不将统计关联解释为因果关系。

## 数据处理

- 数据来源为 `data/raw/form_1.csv`，共 58 条文物记录。
- 分析列固定为 `纹饰`、`类型`、`颜色`、`表面风化`。
- `颜色` 的 4 个缺失值统一映射为字符串 `未知`，保留全部样本。
- 其他分析列如出现缺失值，也使用相同规则标记为 `未知`，并在样本数图中显示。
- 不修改 `form_1.csv`，所有填充仅发生在分析副本中。

## 统计方法

### 列联表

四个变量产生六组无序变量对。每组使用 `pandas.crosstab` 建立观测频数列联表，并由 `scipy.stats.chi2_contingency` 计算：

- Pearson 卡方统计量；
- 自由度；
- 渐近 p 值；
- 期望频数矩阵。

每组结果额外记录最小期望频数，以及期望频数小于 5 的单元格比例。当最小期望频数小于 1，或小于 5 的单元格超过 20% 时，将 `chi_square_reliable` 标记为 `False`。该标记只提示卡方渐近 p 值需谨慎解释，不删除变量对。

### 偏差校正 Cramér’s V

关联强度采用 Bergsma/Wicher 小样本偏差校正形式：

1. 由卡方统计量计算 `phi2 = chi2 / n`；
2. 使用 `max(0, phi2 - ((k-1)(r-1))/(n-1))` 校正；
3. 同时校正有效行列维度；
4. 若校正后的分母不为正，则返回 0；否则返回校正后的 Cramér’s V。

结果范围为 `[0, 1]`。0 表示样本中未观察到关联，越接近 1 表示关联越强。热力图对角线固定为 1，并保持矩阵对称。

## 代码边界

新增 `src/cumcm2022c/analysis/categorical.py`，集中提供：

- `prepare_categorical_data(frame, columns, missing_label="未知")`：复制并规范化分析列；
- `cramers_v_bias_corrected(contingency)`：计算偏差校正 Cramér’s V；
- `pairwise_categorical_associations(frame, columns)`：返回六组检验的整洁 DataFrame；
- `association_matrix(results, columns, value_column)`：将整洁结果转换为对称矩阵。

模块只负责统计计算，不调用绘图库。Notebook 负责图形布局与中文标签，避免统计逻辑只能在单元格中复现。

## Notebook 输出

在 `notebooks/00_data_inspection.ipynb` 现有数据检查之后增加“问题一：分类变量与表面风化”章节，按顺序输出：

1. 四个变量的类别频数表与分面计数柱状图；
2. 六组统计结果表，包括 `variable_1`、`variable_2`、`n`、`chi2`、`dof`、`p_value`、`cramers_v`、`min_expected`、`low_expected_ratio`、`chi_square_reliable`；
3. 4×4 偏差校正 Cramér’s V 热力图，单元格显示三位小数；
4. 4×4 p 值热力图，单元格显示三位有效数字，并在图注说明稀疏表警告；
5. `纹饰`、`类型`、`颜色` 分别相对于 `表面风化` 的三张 100% 堆叠柱状图，横轴为类别、纵轴为类别内部风化比例，并显示每个类别的样本总数。

所有图使用 Matplotlib 与 Seaborn，设置可用的中文字体回退列表；若系统无中文字体，图仍执行，但提示用户安装中文字体。颜色顺序统一为 `无风化`、`风化`。

## 依赖

通过 uv 添加：

- `scipy`：卡方检验；
- `matplotlib`：基础绘图；

## 测试与验证

- 独立变量的列联表得到接近 0 的校正 Cramér’s V。
- 完全一致的两个二分类变量得到 1 的校正 Cramér’s V。
- 结果包含且仅包含六组无序变量对，矩阵对称且对角线为 1。
- 缺失颜色被映射为 `未知`，输入 DataFrame 保持不变。
- 人工构造的稀疏列联表被标记为不可靠。
- 完整测试、Ruff 检查和格式检查通过。
- 使用 `nbconvert --execute` 从头执行数据勘察 Notebook，确认无异常并生成全部表格与图形。

- Notebook 能在 `uv sync` 后从项目根目录完整运行。
